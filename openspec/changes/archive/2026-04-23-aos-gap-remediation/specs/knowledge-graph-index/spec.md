## ADDED Requirements

### Requirement: aos-index.py generates architecture/index.yaml from source files
The script `scripts/aos-index.py` SHALL read all files in `architecture/inbox/` and `architecture/adr/` and produce `architecture/index.yaml`. The index SHALL contain entries keyed by problem ID, ADR ID, and service name, each with their linkages.

#### Scenario: Index is generated from problem and ADR files
- **WHEN** `python scripts/aos-index.py` is run with a populated `inbox/` and `adr/` directory
- **THEN** `architecture/index.yaml` SHALL be created (or overwritten) containing a `graph` key with entries for each problem, ADR, and service referenced

#### Scenario: Problem entry in index contains linked ADRs and services
- **WHEN** `PROB-001.yaml` references services `[api-gateway, billing]` and `ADR-012.md` links to `PROB-001`
- **THEN** the index entry for `PROB-001` SHALL include `linked_adrs: [ADR-012]` and `services: [api-gateway, billing]`

#### Scenario: Index is queryable with yq
- **WHEN** a user runs `yq '.graph | to_entries | .[] | select(.value.services[]? == "billing")' architecture/index.yaml`
- **THEN** the output SHALL contain all problem and ADR entries that reference the `billing` service

### Requirement: Index is regenerated in CI on every push
CI SHALL run `python scripts/aos-index.py` on every push and validate that the resulting `architecture/index.yaml` matches the committed version. A mismatch SHALL emit a warning (non-blocking in Phase 2).

#### Scenario: Committed index is up to date
- **WHEN** the committed `index.yaml` matches what the generator would produce
- **THEN** CI SHALL pass the index freshness check

#### Scenario: Committed index is stale
- **WHEN** new inbox files were added but `index.yaml` was not regenerated before commit
- **THEN** CI SHALL emit a warning identifying the out-of-date index

### Requirement: index.yaml is never hand-edited
`architecture/index.yaml` SHALL be marked as auto-generated. It SHALL not be manually edited. The generator script is the only authoritative source.

#### Scenario: Auto-generated header comment is present
- **WHEN** `architecture/index.yaml` is opened
- **THEN** the first line SHALL contain an auto-generated warning comment such as `# AUTO-GENERATED by scripts/aos-index.py — do not edit manually`
