# ADR-002: Reserve inventory with an atomic stock decrement

## Metadata
- status: accepted
- date: 2026-02-05
- superseded_by: ~
- links: [PROB-002]
- labels: area=fulfillment, criticality=core

## Context

A flash sale oversold 320 units (PROB-002): concurrent requests read the same stock count
before any of them decremented it.

## Decision

Replace read-then-write with a single conditional atomic decrement
(`UPDATE ... SET qty = qty - 1 WHERE sku = ? AND qty > 0`), and hold a short-lived reservation
row while checkout completes. Orders consult `inventory` for the reservation, not raw counts.

## Alternatives

- Global lock on the SKU — rejected (constraint: must not lock the hot path).
- Optimistic versioning with retry — viable but more client complexity than the atomic decrement.

## Consequences

Oversell is eliminated; a sweeper releases expired reservations. A hot SKU still serializes on
its row, which is acceptable at current volume.

## Affected Services
- inventory
- orders
