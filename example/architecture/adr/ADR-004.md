# ADR-004: Dedicated search service with asynchronous indexing

## Metadata
- status: accepted
- date: 2026-03-25
- superseded_by: ~
- links: [PROB-004]
- labels: area=storefront, criticality=supporting

## Context

Search ran `LIKE` scans against `catalog-db` (PROB-004) — slow, and it added load to the
storefront's bottleneck database. New products lagged in results.

## Decision

Stand up `search` over a dedicated `search-index`, fed asynchronously from catalog change
events via `events`. Queries never touch `catalog-db`.

## Alternatives

- Add full-text indexes to `catalog-db` — rejected (more load on the bottleneck).
- Third-party hosted search — deferred (cost; revisit if volume grows).

## Consequences

Search is fast and isolated from catalog writes; results are eventually consistent (seconds of
indexing lag), which is acceptable for discovery.

## Affected Services
- search
