# ADR-005: Redact PII via structured logging

## Metadata
- status: proposed
- date: 2026-04-15
- superseded_by: ~
- links: [PROB-005]
- labels: area=platform, criticality=supporting

## Context

The security audit found emails and order details in plaintext `notifications` logs (PROB-005),
with a mandated 90-day retention.

## Decision

Move to structured logging with a redaction layer that hashes or drops fields tagged as PII
before they reach the log sink. Proposed — open for review.

## Alternatives

- Shorten retention — rejected (constraint: 90 days mandated).
- Scrub logs after the fact — rejected (PII still transits and is briefly stored).

## Consequences

Logs stay useful for debugging without exposing PII. Every log call site must tag PII fields;
a lint rule will enforce it.
